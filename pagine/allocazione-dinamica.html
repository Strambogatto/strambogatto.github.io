💾 Guida Completa all'Allocazione Dinamica

Gestisci la memoria con malloc, calloc e realloc

Cosa è l'Allocazione Dinamica?

L'allocazione dinamica consente di richiedere memoria a runtime. A differenza degli array statici con dimensione fissa, puoi allocare esattamente la quantità di memoria necessaria durante l'esecuzione del programma.

Perché ti serve davvero?

Immagina di scrivere un programma che carica una foto da internet. Non sai quanto pesa finché non la scarichi! O un programma che legge linee di testo da un file - ogni linea potrebbe essere lunga diversa. Con gli array statici dovresti indovinare la dimensione massima e allocare sempre quella, sprechi, sprechi, sprechi. Con la memoria dinamica chiedi esattamente quello che ti serve. Inoltre, ci sono strutture dati (liste, alberi, grafi) che non si possono realizzare senza memoria dinamica. Senza questi concetti, il tuo C rimane limitato a programmi molto semplici.

🎯 Il Problema: Array Statici

Perché gli array normali non bastano

Il limite degli array

In C, quando dichiari un array, devi specificare la dimensione a tempo di compilazione (quando scrivi il codice, non quando lo esegui):

int numeri[100]; // OK, dimensione fissa
int n = 150;
int altro[n]; // ERRORE! n non è una costante

Lo spreco di memoria

Se scrivi un programma che legge i nomi degli studenti di una classe, quanti ne metti? 30? 50? 100? Se scegli 100 ma in una classe ci sono solo 25 studenti, stai sprecando memoria per 75 studenti che non esistono!

#define MAX_STUDENTI 100
char nomi[MAX_STUDENTI][50]; // Spreca memoria per studenti che non esistono

// Meglio:
int num_studenti = 25; // Legguto dal file
char **nomi = malloc(num_studenti * sizeof(char*));

E se non sai la dimensione?

Se scarichi una foto da internet, non sai quanti byte avrà finché non la scarichi completamente. Con array statici è impossibile!

🎪 malloc() - Allocare Memoria

Richiedere memoria al sistema operativo

Come funziona malloc

malloc() chiede al sistema operativo un blocco di memoria di dimensione che tu specifichi. Il sistema operativo trova spazio libero e ritorna un puntatore al primo byte di quel blocco. Il nome "malloc" sta per "memory allocation".

void *malloc(size_t size); // Ritorna void*, puoi assegnarlo a qualsiasi puntatore

Allocare un array di interi

Quanta memoria serve per 50 interi? In C, sizeof(int) ti dice quanti byte occupa un int (solitamente 4):

int num_elementi = 50;
int *arr = (int *)malloc(num_elementi * sizeof(int));
// Alloca 50 * 4 = 200 byte

IMPORTANTE: Controllare se malloc fallisce

Se il sistema operativo non ha abbastanza memoria libera (es. il computer sta per scoppiare di processi aperti), malloc ritorna NULL. Dereferenziare NULL causa un crash. Sempre, SEMPRE controllare!

int *arr = (int *)malloc(50 * sizeof(int));

if (arr == NULL) {
  printf("Errore: memoria insufficiente!\n");
  return 1; // Esci dal programma
}

// Ora puoi usare arr in sicurezza

Cast a (int *)

malloc ritorna void* (puntatore generico). Per assegnarlo a un puntatore specifico, devi fare il cast. In C questo è facoltativo ma buona pratica.

🧹 calloc() - Allocare e Azzerare

Come malloc, ma inizializza a zero

Differenza tra malloc e calloc

calloc fa esattamente quello che fa malloc, ma con un extra: azzera tutti i byte a 0. malloc non inizializza la memoria, quindi contiene spazzatura (valori random da prima).

int *arr1 = (int *)malloc(10 * sizeof(int)); // Contiene spazzatura
int *arr2 = (int *)calloc(10, sizeof(int)); // Tutti zeri

Sintassi di calloc

Nota la differenza: calloc prende due parametri (numero di elementi e dimensione di ogni elemento), mentre malloc ne prende uno (numero totale di byte).

malloc(50 * sizeof(int)); // 50 interi
calloc(50, sizeof(int)); // 50 interi (più leggibile)
calloc(10, sizeof(double)); // 10 double

Quando usare calloc

Usa calloc quando vuoi che la memoria sia inizializzata a zero. È utile per array di numeri (0 è un valore sicuro) o per strutture dati che dipendono dai valori iniziali.

🔄 realloc() - Ridimensionare

Cambiare la dimensione di memoria allocata

Il caso d'uso: crescita dinamica

Immagina di leggere numeri da un file uno per uno. All'inizio pensi di averne 10, poi 20, poi 50... Con realloc puoi ingrandire l'array man mano che necessario.

int *arr = (int *)malloc(10 * sizeof(int));
// Dopo aver riempito i 10 elementi, ne servono altri:
arr = (int *)realloc(arr, 20 * sizeof(int));
// Ora arr ha spazio per 20 interi

Come funziona realloc

realloc fa tre cose:
1. Chiede al sistema operativo un blocco più grande
2. Copia i dati vecchi nel nuovo blocco
3. Libera il blocco vecchio
Il puntatore potrebbe cambiare! Ecco perché DEVI assegnare il risultato:

// SBAGLIATO:
realloc(arr, 20 * sizeof(int)); // Perdi l'indirizzo nuovo!

// CORRETTO:
arr = realloc(arr, 20 * sizeof(int));
if (arr == NULL) { // Sempre controllare!
  printf("Errore in realloc!\n");
  // Ma qual è il vecchio puntatore? Perso!
}

Attenzione al fallimento

Se realloc fallisce, ritorna NULL ma NON libera il blocco vecchio. Per questo, se fai "arr = realloc(...)" e realloc fallisce, perdi il vecchio puntatore. Una strategia migliore:

int *temp = realloc(arr, 20 * sizeof(int));
if (temp == NULL) {
  printf("Errore!\n"); // arr è ancora valido
} else {
  arr = temp; // Solo ora assegniamo
}

🗑️ free() - Liberare Memoria

Ritornare memoria al sistema operativo

Memoria allocata = memoria da liberare

Ogni volta che usi malloc/calloc/realloc, devi OBBLIGATORIAMENTE liberare la memoria quando finisci. Se non lo fai, quella memoria rimane occupata per tutta l'esecuzione del programma. È quello che si chiama "memory leak".

int *arr = (int *)malloc(100 * sizeof(int));
// Usa arr...
free(arr); // Libera la memoria
arr = NULL; // Buona pratica: evita puntatori selvatici

Il pericolo: doppia liberazione

ERRORE COMUNE: liberare la stessa memoria due volte causa crash!

int *arr = (int *)malloc(10 * sizeof(int));
free(arr);
free(arr); // CRASH! Doppia liberazione

Ecco perché è buona pratica mettere NULL dopo free():

free(arr);
arr = NULL; // Se per errore chiamiamo free(arr) di nuovo, non succede nulla
free(arr); // OK, freeing NULL non fa nulla

📊 Array Dinamici: Esempio Pratico

Mettere tutto insieme

Scenario: leggere numeri da tastiera

L'utente ti dice "dammi 5 numeri", poi li digita. Dopo, ti dice "dammi altri 3 numeri". Come li memorizzi? Con un array dinamico che cresce!

int *numeri = NULL;
int capacita = 0;
int lunghezza = 0;

while (1) {
  int n;
  printf("Inserisci numero (0 per finire): ");
  scanf("%d", &n);
  if (n == 0) break;
  
  // Se l'array è pieno, raddoppia la capacità
  if (lunghezza == capacita) {
    capacita = (capacita == 0) ? 10 : capacita * 2;
    numeri = realloc(numeri, capacita * sizeof(int));
    if (numeri == NULL) {
      printf("Errore allocazione!\n");
      return 1;
    }
  }
  
  numeri[lunghezza++] = n;
}

// Stampa
for (int i = 0; i < lunghezza; i++)
  printf("%d ", numeri[i]);

// Libera
free(numeri);
numeri = NULL;

🔗 Liste Collegate: Strutture Dinamiche

Dove la memoria dinamica brilla davvero

Il limite degli array

Gli array dinamici sono flessibili in dimensione, ma non in posizione: inserire o rimuovere un elemento dal mezzo è costoso (devi spostare tutti gli altri). Le liste collegate risolvono questo!

Cos'è una lista collegata?

Una lista collegata è una catena di "nodi". Ogni nodo contiene un dato e un puntatore al nodo seguente. Puoi inserire/rimuovere nodi in O(1) se sai dov'è il punto di inserimento.

struct nodo {
  int dato;
  struct nodo *prossimo;
};

// Creare il primo nodo
struct nodo *testa = (struct nodo *)malloc(sizeof(struct nodo));
testa->dato = 10;
testa->prossimo = NULL;

Inserire un elemento

struct nodo *nuovo = (struct nodo *)malloc(sizeof(struct nodo));
nuovo->dato = 20;
nuovo->prossimo = testa->prossimo; // Inserisci tra testa e il prossimo
testa->prossimo = nuovo;

Traversare la lista

struct nodo *corrente = testa;
while (corrente != NULL) {
  printf("%d ", corrente->dato);
  corrente = corrente->prossimo;
}

⚠️ Memory Leaks e Debugging

Come evitare i comuni errori

Cos'è un memory leak?

Un memory leak è memoria allocata che non viene mai liberata. Se il tuo programma alloca memoria in un loop senza liberarla, consuma sempre più RAM finché il computer non va in crash.

// MEMORY LEAK!
for (int i = 0; i < 1000000; i++) {
  int *arr = (int *)malloc(1000);
  // Usi arr...
  // Dimentichi free(arr);
}
// Dopo il loop, 1 miliardo di byte sono persi!

Regole d'oro

1. Ogni malloc/calloc deve avere un corrispondente free
2. Controlla sempre se malloc ritorna NULL
3. Non libera memoria che non hai allocato
4. Non libera la stessa memoria due volte
5. Non usare memoria dopo aver chiamato free su di essa
6. Se liberi una struttura con puntatori, libera prima i campi

Strumento: valgrind

Su Linux, puoi usare valgrind per trovare memory leaks:

valgrind ./mio_programma
